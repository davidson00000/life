# /idea

1. `NEWS/` ディレクトリから直近2〜3週間分のニュースファイル（`YYYY-MM-DD.md`）を読み込むこと。トリアージセクションがあればそれも参考にすること。
2. ニュースのトレンドや技術動向を分析し、以下の制約に合致するスモールビジネスアイデアを **5件** 提案すること：
   - **年間利益100万円**（月8〜9万円程度）が現実的に見込めるスケール
   - **自分1人 + 複数のAIエージェント**だけで運営できる（従業員不要）
   - AIエージェントが担う役割が明確であること
3. 各アイデアには以下の情報を含めること：
   - **アイデア名**: 一言でわかるタイトル
   - **概要**: 何をどう売るか（2〜3文）
   - **ターゲット**: 誰に売るか
   - **収益モデル**: どう稼ぐか（単価 × 想定顧客数など具体的に）
   - **必要なAIエージェント構成**: どんなエージェントが何を担当するか
   - **初期コスト**: 立ち上げに必要な概算費用
   - **着想元ニュース**: どのニュース記事からヒントを得たか（タイトルとURL）
4. 以下のMarkdownフォーマットで `IDEAS/` ディレクトリに保存すること。ファイル名は `YYYY-MM-DD_ideas.md` とする。

```markdown
# ビジネスアイデア提案: YYYY-MM-DD

> 制約: 年間利益100万円 / 自分1人+AIエージェント運営 / 直近ニュースに基づく

## 1. アイデア名

- **概要**: ...
- **ターゲット**: ...
- **収益モデル**: ...
- **必要なAIエージェント構成**:
  - エージェント1: 役割
  - エージェント2: 役割
- **初期コスト**: ...
- **着想元ニュース**: [記事タイトル](URL)

## 2. アイデア名
...
```

5. ファイル保存後、5件のアイデア名と一言サマリを一覧でユーザーに報告すること。
