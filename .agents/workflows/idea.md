# /idea

1. `IDEAS/YYYY-MM-DD_ideas.md`（本日の日付）が既に存在するか確認する。
   - **存在する場合**: ファイルを読み込み、既出のアイデア名をすべてリストアップする。以降のステップで、これらと同じまたは実質的に同内容のアイデアは提案しないこと。
   - **存在しない場合**: 新規作成する。
2. `NEWS/` ディレクトリから直近2〜3週間分のニュースファイル（`YYYY-MM-DD.md`）を読み込むこと。トリアージセクションがあればそれも参考にすること。
3. ニュースのトレンドや技術動向を分析し、以下の制約に合致するスモールビジネスアイデアを **5件** 提案すること：
   - **年間利益100万円**（月8〜9万円程度）が現実的に見込めるスケール
   - **自分1人 + 複数のAIエージェント**だけで運営できる（従業員不要）
   - AIエージェントが担う役割が明確であること
   - **既出のアイデアと重複しないこと**（同じ切り口・同じターゲット・同じ収益モデルの焼き直しは不可）
4. 各アイデアには以下の情報を含めること：
   - **アイデア名**: 一言でわかるタイトル
   - **概要**: 何をどう売るか（2〜3文）
   - **ターゲット**: 誰に売るか
   - **収益モデル**: どう稼ぐか（単価 × 想定顧客数など具体的に）
   - **必要なAIエージェント構成**: どんなエージェントが何を担当するか
   - **初期コスト**: 立ち上げに必要な概算費用
   - **着想元ニュース**: どのニュース記事からヒントを得たか（タイトルとURL）
5. ファイルへの書き込みルール：
   - ファイルが**新規の場合**: 以下のフォーマットで作成する。
   - ファイルが**既存の場合**: ヘッダー行（`# ビジネスアイデア提案: YYYY-MM-DD` と `> 制約:` 行）の直後に、新しいアイデアのセクションを**先頭に追加**する。番号は通し番号とし、既存アイデアの番号も採番し直すこと（新しいものほど若い番号）。

```markdown
# ビジネスアイデア提案: YYYY-MM-DD

> 制約: 年間利益100万円 / 自分1人+AIエージェント運営 / 直近ニュースに基づく

## 1. アイデア名（← 今回追加分が先頭）

- **概要**: ...
- **ターゲット**: ...
- **収益モデル**: ...
- **必要なAIエージェント構成**:
  - エージェント1: 役割
  - エージェント2: 役割
- **初期コスト**: ...
- **着想元ニュース**: [記事タイトル](URL)

## 6. 以前のアイデア名（← 既存分は番号を繰り下げ）
...
```

6. ファイル保存後、今回新たに追加した5件のアイデア名と一言サマリを一覧でユーザーに報告すること。
